# @kirejs/markdown Documentation for LLMs

## Overview
The Markdown package integrates Markdown rendering capabilities into Kire templates using the `marked` library. It supports rendering Markdown strings, files, and glob patterns.

## Methods
The plugin extends the Kire instance with:
- `kire.mdrender(content, locals)`: Renders a Markdown string and processes it as a Kire template.
- `kire.mdview(path, locals)`: Renders a `.md` file as a Kire template.

## Directives

### `@markdown`
Renders a Markdown file or string.
- **Single File:** `@markdown('docs/intro.md')`
- **Glob Pattern:** `@markdown('blog/*.md')` - Renders all matching files.

**Example:**
```kire
<article>
    @markdown('content/about.md')
</article>
```

### `@mdslots`
Loads multiple Markdown files into a context variable, useful for creating dynamic lists or grids.

**Example:**
```kire
@mdslots('posts/*.md', 'posts')

@for(path in posts)
    <div class="post-preview">
        {{{ posts[path] }}}
    </div>
@end
```

## Usage in Views
If you use `kire.view('my-file.md')`, the plugin automatically handles the resolution and rendering if the extension is recognized.
Note: To use variables inside Markdown, use the standard Kire syntax: `{{ it.variable_name }}`.
