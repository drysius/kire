# @kirejs/components Documentation for LLMs

## Overview
The Components package adds support for a more declarative "Blade-like" component syntax in Kire, allowing you to use custom HTML tags like `<x-component />` instead of the `@wire` or `@include` directives.

## Usage

### Custom Tag Syntax
Any HTML tag starting with `x-` is automatically treated as a Kire component.

**Example:**
```html
<x-button class="btn-primary" type="submit">
    Click Me
</x-button>
```

### Resolution Logic
- The tag name (after `x-`) is mapped to a view path within the configured namespace.
- By default, `<x-user-profile />` looks for a template at `components.user-profile.kire`.
- Dot notation in tags maps to directory structure: `<x-ui.modal />` -> `components.ui.modal.kire`.

### Attributes and Props
- All attributes passed to the tag are injected into the component template as local variables.
- Example: `<x-alert type="danger" />` makes `type` available as `{{ type }}` in the component template.

### Slots
- Any content inside the custom tag is captured and passed to the component as a variable named `slot`.
- **Example:**
  ```html
  <x-card>
      <p>This is my content</p>
  </x-card>
  ```
  Inside `components/card.kire`:
  ```html
  <div class="card">
      {{{ slot }}}
  </div>
  ```

## Configuration
You can define the prefix and the physical path where components are stored.

```typescript
import { KireComponents } from '@kirejs/components';

kire.plugin(KireComponents, {
    prefix: 'ui', // Changes <x-*> to <ui-*> and namespace to 'ui'
    path: './src/views/components' // Physical directory
});
```
