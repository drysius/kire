# @kirejs/route Documentation for LLMs

## Overview
The Route package provides a simple route management helper for Kire templates. It allows checking the current URL and route name within your views.

## Usage
The plugin exposes a global `Route` object to all templates.

### `Route.is(pattern)`
Checks if the current route matches a pattern. Supports wildcards (`*`).

**Example:**
```html
<nav>
    <a href="/" class="{{ Route.is('home') ? 'active' : '' }}">Home</a>
    <a href="/users" class="{{ Route.is('users.*') ? 'active' : '' }}">Users</a>
</nav>
```

### `Route.current()`
Returns the current request path (e.g., `/profile/settings`).

### `Route.currentRouteName()`
Returns the name of the current route if it was set in the manager.

## Integration (Server-side)
You must update the `Route` state in your server-side router before rendering the view.

```typescript
import { Route } from '@kirejs/route';

// In your router middleware
Route.set(req.path, routeName);

// Then render
const html = await kire.render('my-view');
```
